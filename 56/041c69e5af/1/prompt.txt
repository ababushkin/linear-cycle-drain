<title>drain-cycle: no token/time/cost guardrail — kill a worker that breaches limits</title>
<description>
## Problem

The only limit on a worker is a 3600s/issue wall-clock timeout. Nothing caps tokens, cost, or total cycle spend. One issue (<issue id="d9f967fa-c269-4f36-aa5e-0b33391c8b03">ABA-205</issue>) ran 234 turns / 43M tokens / 23 min unchecked, and the cycle had no ceiling at all — five such issues drained the whole quota with no circuit-breaker.

## Change

Add `drain_cycle/limits.py` with thresholds (baked-in defaults, overridable by optional `~/.drain-cycle/limits.yml` and/or CLI flags), and enforce them in the streamed worker:

* **Native belt:** pass `--max-budget-usd <per-issue $cap>` so `claude` self-terminates on cost.
* **Orchestrator suspenders:** from the live event stream, kill the worker's process group when per-issue cumulative tokens or elapsed time breaches; track cycle-wide cumulative tokens / cost / wall-clock and stop the cycle when the cycle cap breaches.
* On breach: process-group kill, then a recorded halt entry whose `halt_reason` names the breached limit and its value at kill time (same exit-1 + revert contract as existing halts).
* Each guardrail independently on/off-able.

Defaults (tunable; calibrate after first real run): per-issue 8M tokens · 20 min · $15; cycle 30M tokens · 90 min · $60. Token count is the primary guardrail (subscription user); cost shown alongside.

## Acceptance criteria

* A worker exceeding the per-issue token cap is killed (whole process group) and produces a halt entry naming the breached limit + value.
* The cycle stops when a cycle-wide cap is breached.
* `--max-budget-usd` is passed to the worker.
* Limits load from defaults, with `limits.yml` override honoured.
* Tests cover each breach path with a fake-claude script that overshoots a tiny cap; full suite green.

## Recommended worker model

**Opus** — kill semantics and breach edge cases.

## Files

`drain_cycle/limits.py` (new), `drain_cycle/worker.py` (+ orchestrator cycle-wide aggregation), `tests/test_limits.py` + worker breach tests, `README.md` (limits + `limits.yml`), `docs/design-decisions.md` (guardrails + defaults decision).

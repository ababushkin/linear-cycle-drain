<title>Live progress meter: omit the cost field (cost is unknown until session end)</title>
<description>
## Problem

The live per-turn progress meter prints `$?.??` for cost on every line, e.g.

```
ABA-276 ¬∑ turn 63 ¬∑ 7.7M tok (peak 149k) ¬∑ $?.?? ¬∑ 5m
```

Cost is only available from the terminal `result` stream event (`total_cost_usd`, parsed in `worker.py:_feed_result`). The per-turn `assistant` events that drive the meter carry token usage but no cost, and the `result` event does **not** re-fire `on_progress` (`worker.py:310` only fires on `type == "assistant"`). So by the time cost exists the session has exited and the meter has stopped printing ‚Äî the meter can never display a real cost on any line, and the `$?.??` placeholder reads as broken.

(Confirmed against `~/.drain-cycle/runs/*.json`: completed sessions record a real `cost_usd`; killed/breached sessions record `None` because the kill lands before the `result` event.)

## Decision

Drop the cost segment from the live meter entirely. Mid-run cost is genuinely unknown, and `drain-cycle status` already reports cost (`‚Äî` when unknown) plus the budget cap from the persisted marker ‚Äî that command stays as-is.

## Target meter line

```
ABA-276 ¬∑ turn 63 ¬∑ 7.7M tok (peak 149k) ¬∑ 5m
```

## Changes

* `drain_cycle/progress.py` ‚Äî `format_progress_line` (\~line 104‚Äì121): remove the cost segment from the returned string. Drop the now-unused `cost_usd` parameter as well ‚Äî once the segment is gone the parameter is dead, and the surgical rule is to remove the orphan this change creates rather than leave a dead arg a future reader has to puzzle over (rejected alternative: keep the signature and silently ignore `cost_usd` ‚Äî smaller diff, but leaves an unused parameter). Update the docstring example (currently `$12.30`).
* `drain_cycle/orchestrator.py` ‚Äî the `format_progress_line(...)` call in `_make_on_progress` (\~line 250‚Äì260): stop passing `cost_usd`. The surrounding `_cb` still writes `cost_usd` into the active marker (\~line 245) so `drain-cycle status` is unaffected.
* No change to `worker.py` (the `on_progress` signature keeps `cost_usd`, which still feeds the marker), to `status.py`, or to the marker-schema docstring (`progress.py:25`) which still documents `cost_usd` in the marker.
* `tests/test_progress.py` ‚Äî the cost-rendering branch is gone, so the two tests that exercise it (`test_format_progress_line_with_cost` ‚Üí asserts `$12.30`, `test_format_progress_line_without_cost` ‚Üí asserts `$?.??`) collapse into one test that asserts the exact cost-free line.

## Acceptance criteria

1. `format_progress_line` returns exactly `IDENT ¬∑ turn N ¬∑ <tok> tok (peak <k>) ¬∑ <elapsed>` (no cost segment, no `$`) ‚Äî asserted against a known input in `test_progress.py`.
2. `drain-cycle status` still prints a `cost:` line (`‚Äî` or value, plus `[cap: $N]`), unchanged.
3. `pytest` is green.

## Out of scope

Client-side running-cost estimation from tokens (the "Option A" path) ‚Äî also the only thing that would give cost for killed/breached sessions, where the final recorded cost is `None` too. Track separately if wanted.

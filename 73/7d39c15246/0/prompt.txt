<issue identifier="ABA-236">
<title>drain-cycle: run logs are token-blind — stream the worker and record usage</title>
<description>
## Problem

The run log records only timestamps and exit code — no tokens, cost, turns, or model. After a run that burned \~108M tokens the operator had no on-disk way to see which issue spent what; usage had to be reconstructed from `~/.claude/projects/*.jsonl`. The worker is also spawned with `subprocess.run(timeout=)` and no `start_new_session`, so on a timeout only the direct child is killed and grandchildren (MCP servers, sub-agents) are orphaned and keep consuming.

## Change

Refactor the spawn into a new `drain_cycle/worker.py` (`run_issue(...)`) that streams and records usage:

* Spawn `claude -p --verbose --output-format stream-json --model <m> --dangerously-skip-permissions <prompt>` via `Popen(..., stdout=PIPE, stderr=STDOUT, text=True, bufsize=1, start_new_session=True)`.
* Parse the event stream line by line. Accumulate per-turn `assistant` `message.usage` (`input_tokens`, `output_tokens`, `cache_creation_input_tokens`, `cache_read_input_tokens`); track cumulative tokens and peak-context (max per-turn input+cache_read+cache_create). Capture the terminal `result` event (`total_cost_usd`, `num_turns`, `session_id`, `is_error`, `usage`). Fall back to accumulated totals if killed before a `result` event.
* `start_new_session=True` + process-group kill (`os.killpg`) on the existing per-issue time cap, so grandchildren are reaped.
* Extend the run-log schema (additive — `grade.py` reads only `cycle_id` + `final_linear_state`): per-entry `model`, `usage` (input/output/cache_create/cache_read/cumulative + peak_context), `cost_usd`, `num_turns`, `session_id`, `is_error`, explicit `duration_seconds`; top-level `cycle_cost_usd`, `cycle_tokens_cumulative`. Update the docstring schema block.

Event schema confirmed against real `claude` v2.1.150 stream-json output (verification spike).

## Acceptance criteria

* A completed issue's run-log entry carries model, cumulative + peak-context tokens, cost_usd, num_turns, session_id.
* `grade` output is unchanged by the new fields.
* A worker that times out has its whole process group killed (no orphaned `claude`/MCP procs — `pgrep -fl claude` clean afterwards).
* Tests use a fake-claude script emitting a scripted stream-json sequence (incl. a `result` event) and a hanging script for the timeout/process-group-kill path; full suite green.

## Recommended worker model

**Opus** — concurrency, signal handling, and stream parsing; the trickiest slice.

## Files

`drain_cycle/worker.py` (new), `drain_cycle/orchestrator.py` (call `worker.run_issue`), `drain_cycle/runlog.py` (schema + aggregates + docstring), `tests/test_worker.py` + run-log tests, `docs/design-decisions.md` (stream-json + Popen + process-group-kill decision).

Foundational: blocks the guardrails, live-status, and plugin-debug slices.

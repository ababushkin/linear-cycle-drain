## Fix (confined to `drain_cycle/linear.py`)

Render a grouped label as `"<group>:<leaf>"` in the wire‚Üíinternal projection that already lives in `pending_issues`, so it is indistinguishable downstream from the existing flat-prefix convention. `repos.py` and `model.py` stay untouched; literal `repo:`/`model:` labels remain backward-compatible.

1. Extend the query selection (`linear.py:200`): `labels { nodes { name parent { name } } }`
2. Add a helper and use it in place of the comprehension at `linear.py:210`:

```python
def _label_name(node: dict[str, Any]) -> str:
    """Render a label node in its prefix form.

    A label in a Linear label *group* arrives as a leaf ``name`` with a
    ``parent`` group label; downstream resolvers match on a ``<group>:``
    prefix (``repo:``, ``model:``), so a grouped label renders as
    ``"<group>:<name>"``. An ungrouped label keeps its bare name, which
    already carries any literal prefix typed into the label name.
    """
    parent = node.get("parent")
    if parent:
        return f"{parent['name']}:{node['name']}"
    return node["name"]
```

```python
issue["labels"] = [_label_name(node) for node in issue["labels"]["nodes"]]
```
